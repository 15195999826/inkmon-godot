## Runtime 素材尺寸标准

默认 tile runtime export 使用 `Global max 512 + fixed pixel density`：

- master/source bake 保留高分辨率：`source_canvas_px=2048`、`source_px_per_unit=512`，用于重导和后续更高档导出。
- runtime tile 默认目标：整套 tile 的最大不透明内容边长不超过 `512px`。
- `512px` 指不透明内容 bbox，不是每张 PNG 强制 `512x512` 画布。
- 允许 tight crop 后 PNG 画布略超过 `512px`，因为需要保留透明 padding 防裁边。例如 `grass_e2` 可以是 `419x516`，其中不透明内容是 `415x512`。
- 同一套 asset set 只计算一个 `runtime_scale`，所有 terrain / elevation 共用；不要按每张图单独撑满 512，否则 `e0/e1/e2` 顶面像素密度会不一致。
- runtime PNG 需要 tight crop 透明边，manifest 中 `size_px` 必须等于实际 PNG 尺寸，`anchor_px` 必须按 crop 后坐标重算。
- manifest 中 `px_per_unit` / `px_per_hex_edge` 写 runtime 后的值；`runtime_export` 保留 source 与 runtime 的完整换算信息。
- 默认颜色层使用 `material_mode=emission`，避免 Blender lighting 把 UV 细节洗软；`lit` 只作为调试/对照模式。

当前硬边 runtime 体积验收：

- `concept-baked`：12 张 tile 合计约 `3.47 MB`；单张约 `217-390 KB`。
- `concept-baked-ink`：12 张 tile 合计约 `3.43 MB`；单张约 `213-385 KB`。
- 这个体积和清晰度作为常用游戏地块默认档可接受；不要盲目升到 768 / 1024。
- 如后续需要图鉴、编辑器近景或宣传截图，再额外导出 high tier，例如 `runtime_768`，不要作为游戏默认素材。

Godot 验收注意：

- 替换 PNG 后必须先跑一次 `godot_console --headless --import --path <repo>`，强制刷新 `.godot/imported/*.ctex`。
- 如果只替换 PNG / manifest 后直接 capture，Godot 可能继续使用旧 imported texture cache，表现为地图被错误拉散。
- reimport 后再跑 `asset_scene.tscn` / `asset_ink_scene.tscn` capture，确认 tile 拼接连续且无 `ERROR`。

## 观察结果

- 硬边 / 倒角 / 宽 rim 的 Godot 拼图截图中，没有看到明显 YSort 错层。
- Blender Freestyle ink 主要增强外轮廓和侧壁立体边，没有明显把内部缝线画脏；是否启用可以后续按审美继续对比。
- 默认倒角参数偏保守；宽 rim 参数 `bevel_inset_world=0.085`、`bevel_drop_world=0.050` 更容易读出顶面边缘。
- concept decor 已生成并接入三条素材场景；后续不要再混用 fable decor 做新候选评审。

## 后续方向

- 如果继续走几何 bake，优先从宽 rim 倒角继续调 Blender 光照、rim 明暗、外框墨线。
- 如果继续走面片，下一步需要做纯白背景 tile 的自动锚定 / 人工拟合工具，把 raw 3D tile 稳定矫正到 manifest 尺寸。
- 所有新素材生图继续使用 `quality=low`，清晰度问题优先通过更高 `size`、bake 分辨率和后处理解决。
