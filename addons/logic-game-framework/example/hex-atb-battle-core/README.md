# Hex ATB Battle Core

## 目录用途

此目录是 **共享数据层（Core Layer）**，存放六边形 ATB 战斗系统的核心数据结构和事件定义。

## 三层架构

本战斗系统采用三层架构设计：

### 1. hex-atb-battle-core（共享数据层）
- **职责**：定义共享的数据结构、事件类型、常量
- **特点**：纯数据，无逻辑，可被逻辑层和表演层共同引用
- **内容**：
  - `events/`：强类型事件定义（BattleEvents）
  - 共享数据结构（坐标、状态等）

### 2. hex-atb-battle（逻辑层）
- **职责**：实现战斗规则、状态管理、AI 决策
- **特点**：纯逻辑，不依赖 Godot 节点系统
- **依赖**：引用 core 层的数据结构和事件

### 3. hex-atb-battle-frontend（表演层）
- **职责**：渲染、动画、音效、UI 交互
- **特点**：依赖 Godot 节点系统，订阅逻辑层事件
- **依赖**：引用 core 层事件，监听逻辑层状态变化

## 设计原则

- **单向依赖**：frontend → battle → core
- **事件驱动**：逻辑层通过事件通知表演层
- **可测试性**：逻辑层独立于渲染，便于单元测试
- **可复用性**：core 层数据结构可被多个系统共享
