## 0. 一句话定位

> 这是一个 ______ 的游戏。（类型 + 最核心的一个钩子）

**（2026-06-10 草拟，可再打磨）**：这是一个**「据点+远征」结构的 InkMon 收集养成自走战**游戏——从据点带队远征野区，把独一无二的野生个体抓回家，养成只属于你的深度英雄队伍。

> **初心（开发者原话，2026-06-11）**："做一个有策略深度的抓宠收集养成游戏。" —— 这是所有设计取舍的质检员：借来的系统不服务这句话就不进来。设计方法 = 有意识的缝合（systems 各有出处，原创性在关节处：观战归功给个体、永久个体×run 远征、地理记忆——皆为缝合处自生的新物种，无单一出处）。

## 1. 玩家在玩什么 / 核心循环

> 一局/一个游戏 session，玩家从头到尾在做什么？哪几件事循环？
> 例：探索世界 → 收集/养成 InkMon → 组队 → 战斗 → 拿奖励 → 再养成。
> 哪一环是"乐趣核心"，哪些是为它服务的支撑环节？

**乐趣核心排序（2026-06-10 grill 敲定）**：

- **第一档（三者并列，都是核）**：**收集**（遇见没见过的 InkMon、搞到手）· **培养**（把它练强）· **构筑队伍**（配出自己的阵容）。
  - ⇒ 收集在第一档意味着：捕捉/野外遭遇/图鉴类基建是**核心投资**，不是锦上添花；获取的仪式感值得花成本。
  - **培养的核心点 = 个体独一无二**（2026-06-10 补）：我的 InkMon 具备独一无二的特点，**哪怕是同一个物种**——"我的这只火狐"和"你的那只火狐"必须不一样。现有个体性来源：①技能槽出生随机 roll ②刻印 ③装备（见 main-game-architecture §8c）；个体性是培养乐趣的根，后续设计扩个体性维度时往这里挂。
- **第二档：羁绊，两层**：① player ↔ InkMon 的关系；② InkMon 彼此之间的关系。是重要感受层，但让位于第一档。
  - （InkMon 之间的关系是首次提出的设计元素，具体机制待后续设计轮展开。）

**宏观形态 = 据点 + 远征（2026-06-10 grill 敲定，三选一：大世界 / 局内循环 / 据点+远征）**：

```
据点（hex 据点：商店/训练/培养/公会/晋升/领养六 NPC）
   │  公会接任务 → 大地图远处揭示任务目标格
   │  选队出发 ←—— 构筑时刻：选 4 只出战 + 带粮/带球/带药
   ▼
远征（世界地图选路，尖塔式单向节点逻辑）：
   每步移动 = 一次选路（近处明牌 / 远处迷雾）→
   野群遭遇 4v4 自走战 → 战后捕捉 / 战利品 →
   补给与 HP 持续消耗（不能回头，贪心走险路还是直奔任务格）
   │  抵达任务格完成任务 → 回据点
   ▼
据点：捕获入库 · 培养永久个体（花钱养成/刻印/装备/进化）· 存档 → 接下一个任务
长线：图鉴点亮 · 勋章段位（gate 更远更险的任务）· 异步 PvP · 地理记忆
```

**远征结构 = 尖塔节点逻辑 × 世界地图表达（2026-06-10 grill 第二轮敲定）**：

- **逻辑层与杀戮尖塔完全一致**：A→B 单向前进、不能回头的有向节点图；表达层换成**连贯世界地图上有真实距离的两点**。比抽象节点图多赚三样：世界连贯性（区域生态 = 收集的空间载体，火山出火系不用解释）、任务副本感（每次任务 ≈ 一个完整副本）、**地理记忆成为玩家成长**（多次远征后"火山在东边"是玩家自己学会的）。
- **任务驱动**：据点公会接任务 → 大地图远处揭示任务目标格 → 走过去的路就是这一局。公会 NPC 从占位升格为任务源头。
- **选路信息 = 近处明牌 + 远处迷雾**：视野内格子明牌标记（野群/事件/资源/精英），视野外迷雾，侦察手段扩视野。钩子：某些 InkMon 个体自带侦察特性 —— 个体独一无二接入探索玩法。
- **前进压力三重**：① 结构性：不能回头（尖塔同款）② 补给钟：每步耗粮、粮尽掉 HP，带多少粮 = 出发前构筑决策 ③ HP 跨战斗不回满（现成机制，adr/0001 carryover）。
- **回程（v1 简化）**：v1 = 主委托完成即可回城（出征结束）；"回程第二段路"（B→A 新分支，FTL 撤离式）留作后期**委托类型**（撤离型委托），不做默认。
- **✅ 世界地理持久（P2 拍板，2026-07-04）**：**开档 `new_game` 时一次生成整张大世界，之后永久固定、进据点档**（data shape，如 `InkMonWorldMapData`；每个存档一个独一无二的世界——"我的档的火山"与"我的这只火狐"同款归属感）。**趟内节点图不固定**：接委托定目标点后，在固定地理上**蔓延生成**这趟的选路 DAG（住 `InkMonMissionState`，transient，丢趟同丢）。**迷雾两层分家**：持久点亮（去过的角落永久点亮大地图，进据点档）/ 趟内视野（本趟节点内容可见性，transient）。遭遇/资源每趟动态刷新。**大地图不进 GI grid 机器**（逻辑真相 = 节点图，hex 只是渲染皮肤；据点/战斗双 grid 现状不动）。v1 减负：数据形状按"生成结果入档"现在定死，世界生成器 v1 极简（或预制图占位），趟内 DAG 生成器 v1 实做，hex 皮肤最简摆；走廊质量/多区域/生成器打磨 = Phase 5+。
- **✅ 存档矛盾已解（P1 拍板，2026-07-04）**：**v1 出征（mission，glossary §4.8）态 = transient，不序列化**（不进 `gi.to_dict`；按 adr/0002 三叉 = GI 持的 transient 对象 `InkMonMissionState`）。据点存档维持 §8 手动多槽不变（已落地零成本）。**出征中禁手动 save/load**——双重理由：① 防 scum；② 防"半态存档"（roster HP/捕获已变但出征态没存，写出精神分裂档）。**出发那一刻自动写一次据点档**（现有 `gi.to_dict`，零新机制）= "丢这趟"的锚点。**出征两出口（v1）**：✅ 主委托完成 → 结算入库回据点；💀 丢这趟 —— 全灭 / 中途退出 / 崩溃走**同一条路径** = 回出发档（全灭惩罚 = 丢整趟收获，零新结算代码；Phase 2 想温和化再议）。不做主动撤退——留给未来"撤离型委托"任务类型，不做常驻按钮。张力免费保住（无档可读），白送"抓到稀有货赶紧回家落袋为安"的撤离压力。**不烧桥**：将来要"退出续玩"再把出征态对象纳入序列化（§5.2 无迁移负担），是加法不是重构。

- **为什么不是大世界（宝可梦式）**：构筑在大世界结构里偏弱（第一档受伤）；内容成本（地图×剧情×节奏）是单人+AI 管线最大风险。
- **为什么不是局内循环（尖塔/云顶式）**：run 重置杀死"个体独一无二"与"羁绊"——第一档三个核死两个。
- **为什么是据点+远征**：四个乐趣核各有家（收集=远征遇见+永久带回家；培养=据点对永久个体投入；构筑=每次出发选队+远征深度逼换思路+PvP 阵容；羁绊=永久陪伴+共同出战的默契挂点）；现有架构白送（活 actor 永久 roster = adr/0001、HP 跨战斗保留、hex 据点+六 NPC 现成）；**不烧桥**——野区铺多、加剧情可长成大世界。

## 2. Battle（你打算先建的部分）

> - 战斗形态：ATB 自走棋？回合制？实时？玩家在战斗里**操作什么**（还是纯自动观战）？
> - 单只 InkMon 的深度：技能数、装备、强化到什么程度？（brief 里写"Dota2 英雄深度"，是认真的吗？）
> - 一场战斗的规模 / 时长 / 胜负条件？
> - battle 跟世界/养成的关系：battle 是目的，还是养成的手段？

**（2026-06-10 grill 敲定的部分）**：

- **玩家在战斗中零干涉（纯观战），定死**。理由：① 自走是身份（§0）；② 曾担心捕捉需要战斗内操作，经设计轮排除——捕捉走**战后捕捉**（见 §3），不需要战斗内干涉；③ 由此录像机制（record-then-playback）及其全部红利（暂停/倍速/跳过/异步 PvP/确定性）原封保留，战斗系统不为捕捉重构。
- **规模规划：4v4**（野外遭遇 = 野群对战，不是单挑）。
- **战斗体验流**：进入战斗 →【主世界冻结】→ 观看回放（可跳过）→ 结果 → **玩家确认离开** → 【主世界恢复】回据点/野区。回放期间世界的钟必须停（架构落点见 main-game-architecture §2②）。
- 单只深度：维持 glossary 5.1"深度英雄（Dota 向）"档位（多技能槽+装备+刻印+勋章+进化）。
- **战斗爽点 = "我打出了我想象中的团战"（想象兑现，2026-06-11 敲定）**：战前脑补 combo → 观战看它兑现/超常发挥。归因模糊不是问题，**超常发挥是 feature**（"这只居然 carry 了"）。推论三条：① 技能交互空间是脑补的原料——**不为可读性简化战斗内深度**；② 呈现层职责 = 让 combo 与高光时刻被看见（visualizer 往"高光强调"方向投入）；③ 自走战把战功归属让给 InkMon 本身（carry 的是它不是玩家操作）→ **观战是个体人格化/羁绊履历的发动机**，超常发挥瞬间 = "我的这只"获得故事的瞬间。

## 3. InkMon 本身（单位 / 养成 / 收集）

> - InkMon 是什么？怎么获得（抓？买？孵？进化？）？
> - 养成维度：等级、属性、元素、技能、装备、刻印、勋章……哪些是真要做的，哪些是噱头？
> - 收集 vs 精养：是"养一队精英"还是"集齐图鉴"？

**获取途径（2026-06-10 grill 敲定框架）**：

- **野外捕捉 = 战后捕捉制（框架锁定）**：4v4 自走战打完，对倒地（downed，HP=0 ≠ 死，贴 adr/0001 死亡语义）的野生个体进行捕捉结算/尝试。**具体形态待定**，候选已讨论过的有：
  - 纯结算：战斗结束系统直接 roll 抓到谁（最简，仪式感弱）；
  - 气绝制：战斗中血量归零概率进入"气绝"状态，战后玩家对气绝个体尝试捕捉；
  - 满注制：击倒方式影响捕获率（打过头惩罚/控制收尾加成）+ 战后逐只亲手投球看摇晃（仪式感最全，战斗↔构筑↔捕捉闭环）。
  - 共同点 = **捕捉永远发生在战斗结束后**，不需要战斗内干涉——这是框架被锁定的原因。
- **领养**：Release-Adopt NPC（现有，据点服务）。
- 个体性见 §1（同物种个体独一无二：技能槽 roll/刻印/装备）。
