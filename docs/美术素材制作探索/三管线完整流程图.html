从 raw 到 tscn 的分叉点

AI

texture-gen 裸模式生图

base image 是几何模板，reference 是 `docs/concept.jpg`。输出先落在 `%APPDATA%\InkMon\Lab\texture-gen\calls\...`，再导出到 repo。

Raw

入 repo 缓冲

`concept素材-v1/raw/*_design_raw.png` 是三条管线共同起点；decor 在 `decor_raw/`。

Process

UV 或 fit

硬边生成 standard paper-net UV；倒角生成 explicit bevel UV；面片不生成 UV，只做 alpha cleanup + fit compose。

Model/Bake

Blender 或跳过

硬边/倒角会在 Blender 里把 UV 贴到模型再 render 成 PNG；面片直接把 full 3D tile 当 sprite。

Godot

asset_scene.tscn

三个场景都用 `art_tile_map_base.gd`，只换 `baked_dir`。最终由 manifest 的 anchor/size 驱动地图拼装。

管线 A：硬边 / `mode2_hard`

目标：把 AI full 3D raw 反投影到 standard paper-net UV，再贴到标准六边形柱体，无 bevel、flat face shading，用来诊断最朴素的几何贴图效果。

AI

生图

`template_design_e0/e1/e2.png` + `docs/concept.jpg`，输出 `raw/*_design_raw.png`。

UV

反投影

`prepare_design_warp_uvs.py` 调 `warp_design_to_uv()`，产出 standard paper-net UV。

Model

硬边柱体

`standard_hex_prism_no_bevel`：无 bevel modifier，无 smooth normals。

Bake

Blender render

`bake_art_concept_uv_set.py --pipeline mode2_hard` 输出 baked sprite。

Godot

asset scene

`codex-硬边-v1/asset_scene.tscn` 读取 `concept-baked/`。

阶段图：raw -> UV -> baked -> tscn

模型结构

mesh_contract: `standard_hex_prism_no_bevel`UV layout: standard paper-net

实际代码路径

blender/scripts/texgen/tile_pipeline_modes.py::MODE2_HARD blender/scripts/bake_assets.py::build_hex_tile blender/scripts/bake_art_concept_uv_set.py

关键参数：`tile_bevel_enabled=false`、`tile_smooth_enabled=false`、`bevel_width=0.0`、`bevel_segments=0`。

管线 B：顶面倒角 / `mode3_top_edge_bevel`

目标：不再把顶面边缘压进同一个 top face，而是给 top edge 显式建 6 条 bevel band。当前主图使用 wide-rim 参数。

AI

生图

复用 `concept素材-v1/raw`，不重新调用 texture-gen。

Fit

拟合 raw

`prepare_beveled_uvs.py` 生成 sidecar，并记录 `origin_px` / polygon fit。

UV

beveled UV

`top + bevel_0..5 + wall_0..5`，wide rim 版本在 `beveled_uv_wide_rim/`。

Model

显式倒角 mesh

18 verts、14 faces：top、6 bevel、6 wall、bottom。

Godot

asset scene

`asset_wide_rim_scene.tscn` 读取 `concept-baked-wide-rim/`。

阶段图：raw -> fit sidecar -> beveled UV -> baked -> tscn

`bevel_inset_world=0.085``bevel_drop_world=0.050`flat normals

管线 C：面片 / full 3D tile patch

目标：最大化保留 AI raw 3D tile 的视觉质量。它不走 Blender tile mesh，不生成 UV；程序只做白底清理、scale/anchor fit，然后 Godot 按 sprite 拼图。

AI

生图

使用同一批 raw 3D tile，raw 本身已经包含顶面、侧墙、体块光感。

Fit

清底与缩放

`prepare_patch_assets.py` flood-fill 白底，使用 fit sidecar 统一 anchor/scale。

Model

无 Blender mesh

这里的“模型感”来自 raw 图；程序不再把它拆成面再贴回 mesh。

Sprite

透明 patch

输出 `assets/baked/tile_*_v0.png`，直接作为 Godot sprite。

Godot

asset scene

`codex-面片-v1/asset_scene.tscn` 读取 `concept素材-v1/assets/baked/`。

阶段图：raw -> fit patch -> tscn

fit 与“模型”职责

flood-fill backgroundGaussian alpha edgefit origin -> `ANCHOR_PX`

实际代码路径

inkmon美术探索/concept素材-v1/tools/prepare_patch_assets.py inkmon美术探索/concept素材-v1/beveled_uv/fit/*_beveled_design_fit.json inkmon美术探索/concept素材-v1/assets/baked/tile_*_v0.png

这条线没有 `UV PNG -> Blender mesh -> bake` 阶段。它的风险点是 raw tile 的外轮廓和 fit 参数；优势是更接近 AI 3D 原图。
