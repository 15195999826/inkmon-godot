# 整图面片 YSort 遮挡标注

[adr/0008](../adr/0008-limited-angle-rotation-two-layer-projection.md) Decision 8 保守主案的配套技术：固定视角下，复杂场景元素用 AI 整图直出，遮挡关系靠手工描几何标注解决——整图不切碎、不出分层素材。

## 机制

- **整图永远垫底**：大图一张 `Sprite2D`，画在所有单位之下（它是地板，从不遮挡任何东西）。
- **遮挡体 = 描出来的"重印章"**：每个会挡住单位的元素（拱门/立柱/断墙/大石）描一个 `Polygon2D`，引用**同一张图**的对应像素（polygon + uv 对齐原图），带**自己的 y 基线**进入 Y-sort 空间，与单位一起排序。
- **排序语义**（每个遮挡体独立判断，整图不参与）：
  - 单位脚 y 在基线下方（更靠前）→ 单位画在遮挡体之后 → 单位站在它前面 ✅
  - 单位脚 y 在基线上方（更靠后）→ 圈内像素重印在单位之上，**只有圈内部分**盖住单位，圈外身体照常可见 → 单位在它后面探出半身 ✅
- 遮挡精确到描的轮廓边缘；一张图描 N 个遮挡体，各管各的。

## 标注规则

1. **一个遮挡体 = 一条基线**。接地走向有变化的物体（细长墙体、L 形转角）按接地走向**分段描**，每段各给基线——拆不拆、在哪拆，人眼看接地线即知，这是该方案适合手工标注的原因。
2. 轮廓只需在"单位可能与之重叠的边界"（顶缘/侧缘）贴紧，内部区域随意覆盖。
3. 描遮挡体与逻辑层标注（格子高度/通行性）是**同一遍人工标注**顺手完成；遮挡体纯视觉，不参与碰撞/寻路。

## Godot 落地

- 父节点开 `y_sort_enabled`；遮挡体 `Polygon2D` 的 `position.y` 即基线（必要时配 `y_sort_origin` 微调）。
- `Polygon2D` 设 `texture` = 整图，`polygon`/`uv` 按原图像素坐标对齐（注意 texture offset，让重印像素与垫底图逐像素重合）。
- 单位（billboard `Sprite2D`/`AnimatedSprite2D`）以脚点 y 参与同一 Y-sort；单位站在高台上时用**屏幕投影 y**（视觉脚点），不是逻辑格 y。

## 配套约束（adr/0008 / glossary 7.3）

- 整图面片必须符合统一出图规格：同固定视角、同光向、像素标尺（px/hex 边）、hex 网格叠加校验。
- 逻辑层对面片覆盖的每个 hex 标格高/通行；视觉边缘**不必**贴合六边形（溢出合法），格子语义清楚即可。
