# 玩法系统实现路线图（gameplay systems roadmap）

> 本文 = 从 [`game-vision.md`](game-vision.md) 的游戏循环推导出的**玩法系统分解 + 实现难度 + 建议顺序**。
> 与 [`main-game-architecture.md`](main-game-architecture.md) 的分工：那边只写**现状真相**，本文写**要去哪、先做什么**。
> 难度评估基于 2026-06-11 代码库现状（统一 live-actor / render2d 统一管线 / 回放冻结世界已落地）；难度是工程性质判断，**不是排期承诺**。内容量（物种/技能/数值）不在本文范围。

---

## 1. 动工前的前置决策（必答，每个一轮短 grill）

| # | 决策 | 为什么必须先做 | 现状 |
|---|---|---|---|
| P1 | **存档语义 vs run 张力** | 远征中任意存读档可废掉"不能回头/丢收获/概率捕捉"全部张力（双评审交叉发现，见 game-vision §1 末）。候选：远征中禁手动存档 / run 内自动快照 / 接受 scum。**决定远征状态的数据形状**，先于远征 flow 实现 | 待 grill |
| P2 | **三地图语境共存** | 据点格 / 远征大地图 / 战斗棋盘三个空间语境在唯一 world GI 里怎么摆（§9 双 grid 临时方案的触发条件已命中："下次改战斗 grid 语义时一并定终态"） | 待 grill |
| P3 | 捕捉形态三选一（纯结算 / 气绝制 / 满注制，见 game-vision §3） | 可推迟到捕捉系统动工时；框架（战后捕捉）已锁定不影响其它系统 | 可缓 |

## 2. 系统分解 × 已有底座 × 难度

| 系统 | 已有的底座 | 缺的部分 | 难度 |
|---|---|---|---|
| 远征地图生成（尖塔 DAG × hex 表达） | hex grid 插件、astar、迷雾可用 render2d 做 | "有向节点图长在 hex 地图上"的生成器：分叉收束、节点类型分布、走廊质量 | **中-高（最重）** |
| 三地图语境共存 | 双 grid 切换（据点/战斗，临时方案） | 三语境归属设计（= P2，设计先于代码） | **中** |
| 任务系统 | NPC handler 模式、command/intent 流（训练战 intent 是现成模板） | 任务数据形状 + 公会 handler 扩展 + 目标格揭示 | 低-中 |
| 野群生成（带个体 roll） | `adopt_unit` 的 roll 机器、species catalog、4v4 用 roster slice 现成 | 按区域生态配野群、临时野生 actor 生命周期 | **低** |
| 战后捕捉（纯结算/气绝档） | downed 语义（HP=0 留 registry）、入队机器（adopt 同款） | 概率公式 + 结算 UI | **低** |
| 战后捕捉（满注档，将来） | **击倒方式数据天然已录在 replay timeline**（record-then-playback 红利：战斗逻辑对捕捉零知觉，捕捉从录像读参数） | 从录像读参数 + 逐只投球交互 | 中 |
| 补给钟 | movement tick 系统 | 每步扣粮 + 粮尽掉血规则 + HUD 一行 | **极低** |
| 远征 flow（出发/途中/回城） | `_replay_active` 冻结同款模式（Host flow 状态 gate 世界泵） | 远征态进出 + 与存档语义（P1）整合 | 低-中 |
| 战斗呈现（combo/高光可见，game-vision §2 想象兑现） | render2d 统一管线、visualizer 槽位制（active 4 / dormant 9，JIT 补） | 逐个补 visualizer + 高光强调演出 | **持续投入型**，单笔都小 |
| 异步 PvP（长线） | 决定性 sim + 录像两端各播，机制天然支持 | 服务端撮合 | 大，正确地排在长线 |

## 3. 三个关键工程判断

1. **远征地图第一版不做程序生成**：手摆 3-5 张固定地图（节点图手工设计），先验证选路体验与决策密度（参照尖塔参数：一次 run 有效决策 10-20 次），生成器等体验参数定型后再写。把最重的工程砍掉一半。
2. **P1/P2 在变贵之前做掉**：都不是重构，是"已知欠着的决定"；写远征代码之后再定会返工。
3. **战斗呈现走 JIT 增量**：visualizer 槽位制天然支持"逻辑层出一个新事件机制 → 补一个三件套"；高光强调（超常发挥/combo 时刻）按 game-vision §2 想象兑现三推论投入，不做一次性大投入。

## 4. 建议实施顺序（顺序可调）

```
Phase 0  前置决策：P1 存档语义 + P2 三地图语境（两轮短 grill）
Phase 1  远征骨架：三地图语境落地 + 远征 flow（出发/回城）+ 手摆 1 张固定地图
         + 补给钟 —— 走通「出发 → 走路 → 回家」
Phase 2  遭遇与捕捉：野群生成 + 4v4 接入远征 + 战后捕捉最简档（P3 此时定）
         —— 走通完整核心循环（可给陌生人玩的最小切片）
Phase 3  任务系统：公会接任务 → 目标格揭示 → 完成判定 —— run 有了名字
Phase 4  选路体验：明牌标记 / 迷雾 / 侦察（含个体侦察特性钩子）+ 节点类型
         （事件 / 资源 / 精英）
Phase 5+ 持续层：战斗高光呈现 · 捕捉满注化 · 图鉴 · 段位 gate · 地图生成器
         · 异步 PvP（长线）
```

> v1 切片定义见 game-vision §2/§3（一据点 + 一张远征图 + 少量任务类型 + 战后捕捉最简档 + 可跳过回放 + 存档）。Phase 2 结束 = 切片可玩。
