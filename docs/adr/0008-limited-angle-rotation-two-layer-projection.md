# 0008: 视角方案与素材投影管线 —— 限角旋转实验 → 保守固定视角拍板

> **状态（2026-06-11 拍板）**：主案 = **保守方案（永久固定视角 + 整图面片），见 Decision 8–10**。Considered Options 与 Decision 1–6 是拍板前同日的激进案（限角旋转）推导与实验结论，**保留为已验证备选**，勿当现行规格读。

延续 [adr/0005](0005-presentation-true-2d-isometric-hex.md)（真 2D 等轴 + AI 烘帧 sprite）。本 ADR 解决其遗留的"统一出图规格（命脉，待定）"，并新增运行时相机自由度的决定。

驱动因素：`inkmon/tools/iso_sandbox/` 实验系列（真实 mon 图 × 可旋转地面）证明——烘死视角的 sprite 在仿射补偿下，小范围 pitch/yaw 内观感良好且数学可控；用户实测后确认"纸片感可接受、限角体验更好"。

## Considered Options

- **A 视角永久固定** — 弃。放弃了沙盘（diorama）立体感，而限角旋转的代价已被实验证明很低。
- **B yaw 60° 步进 + 六方向全量出图** — 弃。hex 六重对称下数学自洽，但全部资产（单位+场景物）×6 出图成本，战棋玩法收益低。
- **C 连续自由旋转** — 弃。单张烘帧图无新视角信息（实验：standee 大角度成纸片侧棱、ground 模式把竖直内容拍平成"地毯熊"），等于推翻 2D 回 3D。
- **限角自由旋转** — 当日早先**选**此项。基准角出图 + 仿射补偿，运行时允许小范围连续 pitch/yaw。

> **同日修正**：素材生产力评估后（无美术人力、AI 整图直出是唯一高产能管线），最终拍板 = **A 的工程化形态**（固定视角 + 整图面片 + YSort 手工标注，Decision 8）为主案；限角自由旋转降为已验证备选。

## Decision

1. **出图基准**：pitch = **35.26°（真等轴）** 烘死；出图 yaw 值**待选**（沙盒滑杆定稿）。世界光固定 —— 相机旋转不动光源，烘焙光影在任何观察角下物理自洽。
2. **运行时范围**（设计参数，沙盒实测调整）：起步建议 yaw ±20°、pitch 30°–44°。失真上界：宽度 ≤6%（cos 20°）、高度 ≤13%（cos 比）。
3. **内容三分投影管线** —— 统一判据：**能像纸盒展开成平面的内容 → 精确投影；展不开的实体 → 立牌近似**。素材 = 展开图（UV 心智模型），"折回去+摆斜"全由引擎仿射完成，素材内不画斜、不画光：
   - **地面层**（tile 顶面 / 底座顶面 / 着地阴影 / 贴花）：**俯视图制作**（无透视）→ ground 公式 `ground_now × ground_baked⁻¹`。对水平平面**严格正确**（圆盘椭圆不变性可证）。
   - **立面层**（tile 侧面 / 崖壁 / 墙等朝向已知的竖直平面）：**正视剖面图制作**（可横向平铺条带）→ 每面各自的精确仿射（水平轴 `ground × d`、竖直轴 `cos(pitch)` 缩放），明暗引擎程序化（`dot(normal, light)`）。平面在正交投影下**严格正确**。±20° yaw 内会露脸的侧面共 **4 个**（前左/前右/左棱/右棱，左右棱在摆动两端轮流出现），逐面差异化 = 按 edge index 换条带；背后 2 面可不出图。MVP 用程序化纯色占位（沙盒现状），升级贴图几何不变。
   - **立牌层**（单位本体 / 树 / 圆滚滚实体）：**固定基准角立绘制作** → standee 公式（图像水平轴随 yaw 转回世界向：`(cos Δyaw, sin Δyaw·sin pitch)`；竖直轴按 `cos(pitch)/cos(pitch₀)` 缩放）。限角内近似良好，轻微纸片感 = 风格（数字桌面 diorama，与棋子底座气质一致）。
   - 平面逼近不了的曲面（圆底座 rim 等）：引擎程序化绘制，或分段平面逼近（同 hex 六段逼近圆）。
4. **资产制作红线：禁止混层单图**。竖立资产烘死视角出图、贴地资产顶面视图出图，各走各的变换。反例（实验实证）：带烘死透视底座的单图熊 —— 底座 rim/椭圆比画死在出图角里，ground 公式下被错误拉伸。**资产不按标准制作才穿帮，方案本身无缺陷**。单位底座（最终未必保留）与着地阴影**不从成图抠**：引擎程序化生成，或按"顶面视图 + 程序化加厚"标准单独制作。
5. **地形管线选型被锁定**：烘死角度的 TileMap 路线（`iso_tilemap_sandbox`）出局；地面层 = 程序化重画或顶面素材投影。
6. **设计红线**：限角旋转定位 = 观赏/手感，**不得成为玩法依赖**（不能出现"必须转视角才能看到"的信息）；地形保持低轮廓或高地做遮挡透明化。
7. **方案分级**：本 ADR §1–6 = **激进方案**（限角旋转）。**保守方案** = 永久固定视角——该案下无重投影，混层单图**合法**（带厚度的地块整图烘死、单位 billboard 直贴），TileMap 烘焙路线（`iso_tilemap_sandbox`）复活为可用载体；但统一出图角度、anchor/painter 排序规格、海拔变体计数（档数 × 地形数）仍是命脉。**素材兼容不对称**：按分层标准制作的素材可无损降级用于保守案（固定视角 = 限角范围为 0 的特例）；整图烘死素材**不可**升级回激进案。
8. **拍板（2026-06-11）：保守方案选为主案**。驱动：无美术人力，AI 整图直出（概念图级质量）是唯一高产能美术管线，而激进案的分层素材标准（俯视图/剖面条带/组合配方）正是图像模型弱项——美术出彩 > 相机自由度。主案组合 = **固定视角 + 整图面片做复杂场景元素（手工标注 YSort 遮挡几何 + 逻辑层标格高/通行）+ 标准地面 tile + 标准地面装饰 + 单位 billboard**。YSort 实现：整图 `Sprite2D` 垫底，遮挡体描 `Polygon2D`（引用同图像素）入 Y-sort 空间配手动 y 基线——遮挡精确到轮廓、每个遮挡体独立排序，细节见 [`docs/reference/ysort-occluder-marking.md`](../reference/ysort-occluder-marking.md)。后果：①固定角度值**重新开放**（不再受 standee 数学绑定 35.26°，概念图感 45°/15° 为候选），沙盒滑杆选定唯一角度后写入 glossary 7.3 永久烘死 ②一致性 QC 成为第一命脉（同角度/光向/比例 + 像素标尺 px/hex 边 + hex 网格叠加校验，不合格重绘）③§1–6 限角旋转 + 三分投影**降为已验证备选**（实验记录保留于 `iso_sandbox`），重启需按分层标准重制素材。
9. **地面渲染 = 自绘**（沿 adr/0007 `render2d`/`iso_hex_grid` 路线，占位多边形换烘焙贴图），TileMap 可用不采用：地图源是逻辑层数据（编辑器绘制/autotile 卖点用不上）、高地块/面片遮挡体/单位需同场 Y-sort（TileMap 混排再踩 `texture_origin`/`y_sort_origin` 坑）、百格级规模无性能诉求。
10. **影子与装饰动态策略**：单位与独立装饰的着地影 = **程序化**（默认从本体素材 alpha 自动生成：剪影压扁 + 朝光向斜切 + 高斯模糊 + 乘半透明黑，离线批量脚本；个别艺术化手绘 override）；整图面片内投影 = 烘进图（光向全局锁定）；标准 tile 只带根部接触暗、**不带长投影**（拼接断裂）；动态影/投墙影不做。**动态三档**：①独立装饰 = canvas shader 摆动（顶点 skew 起步 / fragment UV 弯曲进阶；本体与影子共享 time + 每实例随机相位，影子幅度打折）②复杂动态（瀑布/旗帜/水面）= 视频切帧叠加，同单位管线 ③**整图面片 = 静态专属**，想动的元素一律拆成独立装饰叠加——反向成为出图规则（面片 prompt 不含动态元素）。

## Consequences

- glossary 7.3"统一出图规格"：素材量产标准已随 Decision 8 拍板**跟保守案**（整图固定视角）；仍开放：唯一固定角度值（沙盒选定）、光向、比例/像素标尺细节。
- 单位朝向（六方向/镜像）与相机旋转**解耦**，是独立的后续决定（B 案出局后不再强制六方向）。
- 实验载体与公式实现：`inkmon/tools/iso_sandbox/iso_hex_prism_renderer.gd`（`_unit_screen_transform` 三模式）+ `iso_angle_sandbox.tscn`（拆层熊 vs 单图熊对照、unit_mode 按钮）；投影纯函数 `inkmon/presentation/render2d/core/iso_projection.gd`。
- 难逆转点：出图规格 + 地面层管线选型 —— 素材量产开始后变更成本极高。
